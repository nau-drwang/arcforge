# ArcForge Cloudflare Pages Full Stack Deployment

This project uses one GitHub repo with separated frontend/backend folders:

- Frontend source: `frontend/src/` and `frontend/public/`
- Backend source: `backend/functions/`
- D1 migrations: `backend/migrations/`
- Generated frontend output: `dist/`
- Generated Cloudflare Functions mirror: `functions/`

Cloudflare Pages still deploys from the single repo.

## Cloudflare Pages settings

Build command:

```bash
npm ci && npm run pages:build
```

Build output directory:

```text
dist
```

Root directory:

```text
leave blank
```

Node.js version:

```text
22
```

Compatibility flags, if needed:

```text
nodejs_compat
```

## Bindings

Add these in Cloudflare Pages -> Settings -> Functions -> Bindings.

```text
D1 database binding
Variable name: DB
Database: arcforge-database
```

```text
R2 bucket binding
Variable name: MEDIA
Bucket: arcforge-media
```

## Environment variables

Required for the admin portal:

```text
ADMIN_PASSWORD = your private admin password
```

Optional:

```text
ADMIN_SESSION_SECRET = a long random string for signing admin cookies
```

Redeploy after adding or changing environment variables.

## Initialize D1

Open the D1 database console for `arcforge-database` and execute the SQL files in this order:

```text
backend/migrations/0000_groovy_mongu.sql
backend/migrations/0001_arcforge_artwork_metadata.sql
```

Then visit `/admin.html`, log in, and click **Import seed** once if artwork records have not been populated yet.

## Development rule

Edit the canonical source folders only:

- Frontend: `frontend/`
- Backend: `backend/`
- Seed/content: `data/`
- Docs: `docs/`

Do not manually edit generated `dist/` or root `functions/` unless you are applying an emergency deployment patch. They are regenerated by `npm run build`.
